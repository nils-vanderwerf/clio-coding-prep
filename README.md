# Clio Programming Interview — Practice Repo

## Interview Format
- 60 minutes via Zoom + CoderPad
- Ruby (your choice — pick what you know best)
- Console application — no framework, no Rails
- RSpec expected
- No AI tools during the interview
- **They want to see how you think, not just what you produce — talk out loud constantly**

---

## The Golden Rules (read before every session)

1. **Say your approach out loud before typing anything**
2. **Ask yourself: what is the key? what is the value? which method?**
3. **Get it working first — refactor after**
4. **When stuck, say what you know, not what you don't**
5. **Edge cases: empty input, nil, duplicates, negative numbers**

## How to Use CoderPad

Practice in https://coderpad.io/sandbox — same environment as the interview.
- Ruby is available, RSpec is available
- Run code with the play button
- Talk out loud as you type — get used to the format

## What They're Evaluating

- **Problem solving** — can you break a problem down?
- **Communication** — do you think out loud?
- **Ruby fluency** — do you know the standard library?
- **Testing** — can you write meaningful RSpec?
- **Collaboration** — do you ask good questions?
